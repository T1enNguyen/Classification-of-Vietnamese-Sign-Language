# Lý thuyết mạng ResNet50 (Residual Network 50)

## I. Giới thiệu về ResNet

### 1. Bối cảnh ra đời

ResNet (Residual Network) được giới thiệu bởi Kaiming He và cộng sự từ Microsoft Research trong bài báo "Deep Residual Learning for Image Recognition" tại hội nghị CVPR 2015. ResNet đã giành chiến thắng tại cuộc thi ImageNet Large Scale Visual Recognition Challenge (ILSVRC) 2015 với tỷ lệ lỗi top-5 chỉ còn 3.57%, vượt trội so với các phương pháp trước đó.

### 2. Vấn đề của mạng sâu truyền thống

Trước khi ResNet ra đời, các nhà nghiên cứu gặp phải một nghịch lý:

**Vấn đề Degradation (Suy giảm):**
- Khi tăng độ sâu của mạng neural, độ chính xác không tăng mà còn giảm
- Điều này không phải do overfitting (vì cả training error và test error đều tăng)
- Nguyên nhân: Vanishing/Exploding Gradients và khó khăn trong tối ưu hóa

**Ví dụ minh họa:**
```
Mạng 20 layers: Training error = 15%, Test error = 20%
Mạng 56 layers: Training error = 25%, Test error = 30%
→ Mạng sâu hơn nhưng kết quả tệ hơn!
```

### 3. Ý tưởng cốt lõi của ResNet

ResNet giải quyết vấn đề này bằng cách giới thiệu **Residual Learning** (Học phần dư):

**Thay vì học trực tiếp ánh xạ H(x), mạng học phần dư F(x) = H(x) - x**

```
Cách truyền thống: H(x) = f(x)
Cách ResNet: H(x) = F(x) + x
Trong đó: F(x) là phần dư cần học
```

**Lợi ích:**
- Nếu ánh xạ tối ưu gần với identity mapping (H(x) ≈ x), mạng chỉ cần học F(x) ≈ 0
- Dễ tối ưu hơn nhiều so với học toàn bộ H(x)
- Gradient có thể truyền trực tiếp qua skip connections

## II. Kiến trúc ResNet50

### 1. Tổng quan

ResNet50 là một biến thể của họ ResNet với **50 layers** (lớp có trọng số), bao gồm:
- 1 Convolutional layer đầu tiên
- 48 Convolutional layers trong các Residual Blocks
- 1 Fully Connected layer cuối cùng

**Cấu trúc tổng thể:**
```
Input (224×224×3)
    ↓
Conv1 (7×7, 64, stride 2)
    ↓
Max Pooling (3×3, stride 2)
    ↓
Conv2_x (3 blocks) - 256 channels
    ↓
Conv3_x (4 blocks) - 512 channels
    ↓
Conv4_x (6 blocks) - 1024 channels
    ↓
Conv5_x (3 blocks) - 2048 channels
    ↓
Global Average Pooling
    ↓
Fully Connected (1000 classes)
    ↓
Softmax
```

### 2. Residual Block (Bottleneck Design)

ResNet50 sử dụng **Bottleneck Block** để giảm số lượng tham số:

```
Input (H×W×C)
    ↓
┌─────────────────────────┐
│  1×1 Conv (C/4)         │  ← Giảm chiều
│  Batch Norm + ReLU      │
│         ↓               │
│  3×3 Conv (C/4)         │  ← Xử lý chính
│  Batch Norm + ReLU      │
│         ↓               │
│  1×1 Conv (C)           │  ← Tăng chiều
│  Batch Norm             │
└─────────────────────────┘
         ↓
    F(x) + x  ← Skip Connection
         ↓
      ReLU
         ↓
   Output (H×W×C)
```

**Giải thích:**
- **1×1 Conv đầu tiên:** Giảm số kênh xuống 1/4 để giảm tính toán
- **3×3 Conv:** Thực hiện convolution chính trên số kênh đã giảm
- **1×1 Conv cuối:** Tăng số kênh trở lại để khớp với input
- **Skip Connection:** Cộng trực tiếp input vào output

### 3. Chi tiết các stage

| Stage | Output Size | Bottleneck Blocks | Số lượng blocks |
|-------|-------------|-------------------|-----------------|
| Conv1 | 112×112 | 7×7, 64, stride 2 | 1 |
| Conv2_x | 56×56 | [1×1, 64]<br>[3×3, 64]<br>[1×1, 256] | ×3 |
| Conv3_x | 28×28 | [1×1, 128]<br>[3×3, 128]<br>[1×1, 512] | ×4 |
| Conv4_x | 14×14 | [1×1, 256]<br>[3×3, 256]<br>[1×1, 1024] | ×6 |
| Conv5_x | 7×7 | [1×1, 512]<br>[3×3, 512]<br>[1×1, 2048] | ×3 |
| | 1×1 | Global Average Pooling | 1 |
| | | Fully Connected 1000 | 1 |

**Tổng số layers có trọng số:**
- Conv1: 1 layer
- Conv2_x: 3 blocks × 3 layers = 9 layers
- Conv3_x: 4 blocks × 3 layers = 12 layers
- Conv4_x: 6 blocks × 3 layers = 18 layers
- Conv5_x: 3 blocks × 3 layers = 9 layers
- FC: 1 layer
- **Tổng: 1 + 9 + 12 + 18 + 9 + 1 = 50 layers**

## III. Skip Connection (Shortcut Connection)

### 1. Hai loại Skip Connection

#### a) Identity Shortcut (Khi kích thước không đổi)

```python
# Pseudo code
def residual_block(x):
    F_x = conv_block(x)  # F(x)
    output = F_x + x      # F(x) + x
    output = relu(output)
    return output
```

**Điều kiện:** Input và output có cùng kích thước (H×W×C)

#### b) Projection Shortcut (Khi kích thước thay đổi)

```python
# Pseudo code
def residual_block_projection(x):
    F_x = conv_block(x)           # F(x)
    x_projected = conv_1x1(x)     # Chiếu x về kích thước mới
    output = F_x + x_projected    # F(x) + W_s·x
    output = relu(output)
    return output
```

**Điều kiện:** Khi số kênh hoặc kích thước không gian thay đổi

**Ví dụ:**
- Input: 56×56×256
- Output: 28×28×512
- Cần projection: 1×1 Conv với stride 2 để khớp kích thước

### 2. Lợi ích của Skip Connection

✅ **Giải quyết Vanishing Gradient:**
```
Gradient flow: ∂L/∂x = ∂L/∂y · (∂F/∂x + 1)
                              ↑
                    Luôn có thành phần = 1
```
Gradient luôn có thể truyền trực tiếp qua skip connection

✅ **Dễ tối ưu hóa:**
- Nếu layer không cần thiết, mạng có thể học F(x) = 0
- Khi đó H(x) = x (identity mapping)

✅ **Cho phép huấn luyện mạng rất sâu:**
- ResNet đã thử nghiệm thành công với 152, 200, thậm chí 1000+ layers

## IV. Batch Normalization trong ResNet

### 1. Vai trò

Batch Normalization được áp dụng sau mỗi Convolutional layer và trước activation function:

```
Conv → Batch Norm → ReLU
```

### 2. Công thức

Cho một batch với N samples:

**Bước 1: Tính mean và variance**
```
μ_B = (1/N) Σ x_i
σ²_B = (1/N) Σ (x_i - μ_B)²
```

**Bước 2: Normalize**
```
x̂_i = (x_i - μ_B) / √(σ²_B + ε)
```

**Bước 3: Scale and shift**
```
y_i = γ·x̂_i + β
```

Trong đó γ và β là các tham số học được.

### 3. Lợi ích

✅ Tăng tốc độ huấn luyện

✅ Cho phép sử dụng learning rate cao hơn

✅ Giảm sự phụ thuộc vào khởi tạo trọng số

✅ Có tác dụng regularization nhẹ

## VI. Transfer Learning với ResNet50

### 1. Khái niệm Transfer Learning

Transfer Learning là kỹ thuật sử dụng mô hình đã được huấn luyện trên một tập dữ liệu lớn (như ImageNet) và tinh chỉnh (fine-tune) cho bài toán mới.

**Lý do hiệu quả:**
- Các layer đầu học các đặc trưng cơ bản (edges, textures, colors)
- Các layer giữa học các đặc trưng trung gian (shapes, patterns)
- Các layer cuối học các đặc trưng cụ thể cho task

### 2. Quy trình Transfer Learning

#### Bước 1: Load Pre-trained Model

```python
from tensorflow.keras.applications import ResNet50

# Load mô hình đã huấn luyện trên ImageNet
base_model = ResNet50(
    weights='imagenet',      # Sử dụng trọng số từ ImageNet
    include_top=False,       # Bỏ lớp phân loại cuối
    input_shape=(224, 224, 3)
)
```

#### Bước 2: Freeze Base Model

```python
# Đóng băng toàn bộ base model
base_model.trainable = False
```

**Lý do:** Giữ nguyên các đặc trưng đã học được từ ImageNet.

#### Bước 3: Thêm Custom Head

```python
from tensorflow.keras import layers, Model

# Thêm các layer mới cho bài toán cụ thể
x = base_model.output
x = layers.GlobalAveragePooling2D()(x)
x = layers.Dropout(0.4)(x)
predictions = layers.Dense(num_classes, activation='softmax')(x)

model = Model(inputs=base_model.input, outputs=predictions)
```

#### Bước 4: Train Custom Head (Warmup)

```python
model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# Huấn luyện 5-10 epochs
history = model.fit(train_data, validation_data=val_data, epochs=5)
```

#### Bước 5: Fine-tune (Optional)

```python
# Rã đông một số layer cuối của base model
base_model.trainable = True

# Đóng băng các layer đầu, chỉ fine-tune layer cuối
for layer in base_model.layers[:-30]:
    layer.trainable = False

# Compile lại với learning rate nhỏ hơn
model.compile(
    optimizer=tf.keras.optimizers.SGD(learning_rate=1e-5, momentum=0.9),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# Tiếp tục huấn luyện
history_finetune = model.fit(train_data, validation_data=val_data, epochs=10)
```

### 3. Chiến lược Fine-tuning

| Chiến lược | Khi nào sử dụng | Learning Rate |
|------------|-----------------|---------------|
| **Freeze toàn bộ** | Dataset nhỏ, tương tự ImageNet | 1e-3 |
| **Fine-tune layer cuối** | Dataset trung bình | 1e-4 đến 1e-5 |
| **Fine-tune toàn bộ** | Dataset lớn, khác ImageNet | 1e-5 đến 1e-6 |

### 4. Lợi ích của Transfer Learning

✅ **Tiết kiệm thời gian:** Không cần huấn luyện từ đầu

✅ **Cần ít dữ liệu hơn:** Tận dụng kiến thức từ ImageNet

✅ **Độ chính xác cao hơn:** Đặc biệt với dataset nhỏ

✅ **Tránh overfitting:** Pre-trained weights là regularization tốt

## VIII. Công thức toán học chi tiết

### 1. Forward Pass của Residual Block

Cho input **x**, output **y** của một residual block:

```
y = F(x, {W_i}) + x
```

Trong đó:
- **F(x, {W_i})** là residual mapping cần học
- **{W_i}** là tập các trọng số của các layer trong block
- **x** là identity mapping (skip connection)

Với bottleneck block:
```
F(x) = W_3 · σ(W_2 · σ(W_1 · x))
```

Trong đó:
- **W_1**: 1×1 Conv (giảm chiều)
- **W_2**: 3×3 Conv (xử lý chính)
- **W_3**: 1×1 Conv (tăng chiều)
- **σ**: Activation function (ReLU)

### 2. Backward Pass (Gradient Flow)

Gradient của loss L đối với x:

```
∂L/∂x = ∂L/∂y · ∂y/∂x
      = ∂L/∂y · (∂F/∂x + ∂x/∂x)
      = ∂L/∂y · (∂F/∂x + I)
```

Trong đó **I** là identity matrix.

**Ý nghĩa:** Gradient luôn có thành phần identity, đảm bảo gradient không bị vanish.

### 3. Số lượng tham số

Cho một bottleneck block với input channels C_in và output channels C_out:

```
Params = (C_in × C_out/4 × 1 × 1)     # 1×1 Conv đầu
       + (C_out/4 × C_out/4 × 3 × 3)  # 3×3 Conv
       + (C_out/4 × C_out × 1 × 1)    # 1×1 Conv cuối
       
       = C_in × C_out/4 + 9 × (C_out/4)² + C_out²/4
```

**Ví dụ:** Block với C_in = 256, C_out = 256:
```
Params = 256 × 64 + 9 × 64² + 64 × 256
       = 16,384 + 36,864 + 16,384
       = 69,632 parameters
```

🎯 **Giải quyết degradation problem:** Cho phép huấn luyện mạng rất sâu

✅ **Cộng đồng lớn:** Nhiều tài liệu, tutorial, và support
