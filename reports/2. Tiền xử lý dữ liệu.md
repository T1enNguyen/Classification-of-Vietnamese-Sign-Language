# Tiền xử lý dữ liệu (Data Preprocessing)

## I. Tổng quan về quy trình tiền xử lý

Tiền xử lý dữ liệu là bước quan trọng trong việc chuẩn bị dữ liệu cho mô hình Deep Learning. Đối với bài toán phân loại ký hiệu ngôn ngữ ký hiệu Việt Nam, quy trình tiền xử lý bao gồm các bước sau:

1. **Xử lý dữ liệu trùng lặp**
2. **Tách dữ liệu (Train-Validation-Test Split)**
3. **Lưu trữ dữ liệu tối ưu (TFRecord Format)**
4. **Tăng cường dữ liệu (Data Augmentation)**
5. **Chuẩn hóa dữ liệu (Normalization)**

---

## II. Xử lý dữ liệu trùng lặp

### 1. Phát hiện vấn đề

Sau khi thực hiện EDA (Exploratory Data Analysis), phát hiện rằng lớp **AAA** (Ă) có dữ liệu trùng lặp hoặc không phù hợp với mục tiêu nghiên cứu. Quyết định loại bỏ lớp này để đảm bảo chất lượng dữ liệu.

### 2. Thực hiện xóa lớp AAA

```python
import shutil
import os

# Đường dẫn tới thư mục class AAA
class_to_remove = 'AAA'
path_to_remove = os.path.join(data_dir, class_to_remove)

# Xóa thư mục
shutil.rmtree(path_to_remove)
print(f"Đã xóa thành công thư mục: {path_to_remove}")

# Cập nhật lại danh sách classes
classes = sorted([d for d in os.listdir(data_dir) if os.path.isdir(os.path.join(data_dir, d))])
```

### 3. Kết quả sau khi xử lý

- **Số lớp ban đầu:** 26 lớp
- **Số lớp sau khi xử lý:** 25 lớp
- **Tổng số mẫu:** 25,000 ảnh (1,000 ảnh/lớp)
- **Danh sách lớp:** A, AA, B, C, D, DD, E, G, H, I, K, L, M, N, O, OOO, P, Q, R, S, T, U, V, X, Y

**Lý do loại bỏ:** Đảm bảo tính nhất quán của dữ liệu và tránh sự nhầm lẫn giữa các lớp tương tự (AA, AAA).

## III. Tách dữ liệu (Train-Validation-Test Split)

### 1. Mục tiêu

Chia dữ liệu thành 3 tập:
- **Train (80%):** Dùng để huấn luyện mô hình
- **Validation (10%):** Dùng để điều chỉnh hyperparameters và theo dõi quá trình huấn luyện
- **Test (10%):** Dùng để đánh giá hiệu suất cuối cùng của mô hình

### 2. Phương pháp thực hiện

Sử dụng TensorFlow Dataset API để tách dữ liệu một cách hiệu quả:

```python
import tensorflow as tf

# Thiết lập các tham số
BATCH_SIZE = 32
IMG_SIZE = (224, 224)
SEED = 42

# Load toàn bộ tập dữ liệu
full_dataset = tf.keras.utils.image_dataset_from_directory(
    data_dir,
    shuffle=True,
    seed=SEED,
    image_size=IMG_SIZE,
    batch_size=BATCH_SIZE
)

# Tính toán số lượng batch
dataset_size = tf.data.experimental.cardinality(full_dataset).numpy()
train_size = int(0.8 * dataset_size)
val_test_size = dataset_size - train_size
val_size = int(0.5 * val_test_size)
test_size = val_test_size - val_size

# Thực hiện tách
train_ds = full_dataset.take(train_size)
remaining_ds = full_dataset.skip(train_size)
val_ds = remaining_ds.take(val_size)
test_ds = remaining_ds.skip(val_size)
```

### 3. Kết quả phân chia

| Tập dữ liệu | Số batches | Số mẫu | Tỷ lệ |
|-------------|-----------|---------|-------|
| **Train** | 625 | 20,000 | 80% |
| **Validation** | 78 | 2,496 | ~10% |
| **Test** | 79 | 2,504 | ~10% |
| **Tổng** | 782 | 25,000 | 100% |

### 4. Phân bố số lượng mẫu theo lớp

Sau khi tách dữ liệu, kiểm tra phân bố số lượng mẫu trên mỗi lớp để đảm bảo tính cân bằng:

#### Tập Train (20,000 mẫu)
Mỗi lớp có từ 770-819 mẫu, phân bố tương đối đồng đều.

#### Tập Validation (2,496 mẫu)
Mỗi lớp có từ 75-115 mẫu, đảm bảo đủ dữ liệu để đánh giá trong quá trình huấn luyện.

#### Tập Test (2,504 mẫu)
Mỗi lớp có từ 82-122 mẫu, đủ để đánh giá hiệu suất tổng quát của mô hình.

**Nhận xét:** Phân bố dữ liệu sau khi tách vẫn giữ được tính cân bằng giữa các lớp, không có lớp nào bị thiếu hụt dữ liệu nghiêm trọng.

## IV. Lưu trữ dữ liệu tối ưu

### 1. So sánh các phương pháp lưu trữ

#### Phương pháp 1: TensorFlow Dataset.save()

```python
# Lưu dữ liệu
train_ds.save('/path/to/train_ds')
val_ds.save('/path/to/val_ds')
test_ds.save('/path/to/test_ds')
```

**Kết quả:**
- Train Dataset: 11,484.49 MB
- Val Dataset: 1,433.26 MB
- Test Dataset: 1,437.86 MB
- **Tổng dung lượng: ~14.36 GB**

**Phân tích:**
- Mỗi ảnh 224×224×3 (float32) chiếm: 224 × 224 × 3 × 4 bytes ≈ 0.57 MB
- Tổng 25,000 ảnh: 25,000 × 0.57 MB ≈ 14.25 GB
- Dung lượng thực tế khớp với lý thuyết

**Nhược điểm:** Dung lượng quá lớn, không phù hợp để lưu trữ trên Google Drive hoặc chia sẻ.

#### Phương pháp 2: TFRecord với nén GZIP (Được chọn)

```python
def serialize_example(image, label):
    """Tạo một tf.train.Example message từ (image, label)"""
    # Mã hóa ảnh thành JPEG để tiết kiệm dung lượng
    image = tf.cast(image, tf.uint8)
    image_encoded = tf.io.encode_jpeg(image)
    
    feature = {
        'image': _bytes_feature(image_encoded),
        'label': _int64_feature(label),
    }
    example_proto = tf.train.Example(features=tf.train.Features(feature=feature))
    return example_proto.SerializeToString()

def save_dataset_as_tfrecord(dataset, filename):
    options = tf.io.TFRecordOptions(compression_type="GZIP")
    with tf.io.TFRecordWriter(filename, options=options) as writer:
        for image_batch, label_batch in dataset:
            for i in range(len(image_batch)):
                example = serialize_example(image_batch[i], label_batch[i])
                writer.write(example)
```

**Kết quả:**
- train.tfrecord: 250.13 MB
- val.tfrecord: 31.31 MB
- test.tfrecord: 31.55 MB
- **Tổng dung lượng: ~313 MB**

**Ưu điểm:**
- Giảm dung lượng **45.8 lần** (từ 14.36 GB xuống 313 MB)
- Hỗ trợ nén GZIP tích hợp
- Định dạng tối ưu cho TensorFlow
- Dễ dàng chia sẻ và lưu trữ trên cloud

### 2. Đọc dữ liệu từ TFRecord

```python
feature_description = {
    'image': tf.io.FixedLenFeature([], tf.string),
    'label': tf.io.FixedLenFeature([], tf.int64),
}

def _parse_function(example_proto):
    # Giải mã một example
    parsed_features = tf.io.parse_single_example(example_proto, feature_description)
    
    # Giải mã ảnh JPEG
    image = tf.io.decode_jpeg(parsed_features['image'], channels=3)
    image = tf.image.resize(image, [224, 224])
    
    label = tf.cast(parsed_features['label'], tf.int32)
    return image, label

def load_tfrecord_dataset(tfrecord_path, batch_size=32):
    raw_dataset = tf.data.TFRecordDataset(tfrecord_path, compression_type="GZIP")
    parsed_dataset = raw_dataset.map(_parse_function)
    return parsed_dataset.batch(batch_size).prefetch(buffer_size=tf.data.AUTOTUNE)
```

## V. Tăng cường dữ liệu (Data Augmentation)

### 1. Mục tiêu

Data Augmentation giúp:
- Tăng tính đa dạng của dữ liệu huấn luyện
- Giảm overfitting
- Cải thiện khả năng tổng quát hóa của mô hình
- Mô phỏng các điều kiện thực tế (góc chụp, ánh sáng khác nhau)

### 2. Các kỹ thuật được áp dụng

```python
def augment_data(image, label):
    """Áp dụng các phép biến đổi ngẫu nhiên cho ảnh"""
    # Lật ngang ngẫu nhiên
    image = tf.image.random_flip_left_right(image)
    
    # Thay đổi độ sáng ngẫu nhiên (±20%)
    image = tf.image.random_brightness(image, max_delta=0.2)
    
    # Thay đổi độ tương phản ngẫu nhiên (80%-120%)
    image = tf.image.random_contrast(image, lower=0.8, upper=1.2)
    
    return image, label

# Áp dụng augmentation cho tập train
train_ds_augmented = train_ds.map(augment_data, num_parallel_calls=tf.data.AUTOTUNE)
```

### 3. Chi tiết các kỹ thuật

| Kỹ thuật | Tham số | Mục đích |
|----------|---------|----------|
| **Random Flip Left-Right** | 50% xác suất | Tăng tính đa dạng về hướng cử chỉ tay |
| **Random Brightness** | max_delta=0.2 | Mô phỏng điều kiện ánh sáng khác nhau |
| **Random Contrast** | lower=0.8, upper=1.2 | Tăng khả năng nhận diện trong điều kiện tương phản thấp |

### 4. Lưu ý quan trọng

- **Chỉ áp dụng cho tập Train:** Tập Validation và Test không được augmentation để đảm bảo đánh giá chính xác
- **Không sử dụng rotation:** Vì góc quay của cử chỉ tay có ý nghĩa ngữ nghĩa, xoay ảnh có thể làm thay đổi ý nghĩa của ký hiệu
- **Không crop quá mức:** Để tránh mất thông tin quan trọng về hình dạng bàn tay

## VI. Chuẩn hóa dữ liệu (Normalization)

### 1. Preprocessing cho ResNet50

Sử dụng hàm `preprocess_input` của ResNet50 để chuẩn hóa dữ liệu theo cách mà mô hình đã được huấn luyện trên ImageNet:

```python
from tensorflow.keras.applications.resnet50 import preprocess_input

def preprocess_for_resnet(image, label):
    """Chuẩn hóa ảnh theo chuẩn ResNet50"""
    image = preprocess_input(image)
    return image, label

# Áp dụng preprocessing
train_ds_processed = train_ds_augmented.map(preprocess_for_resnet, 
                                            num_parallel_calls=tf.data.AUTOTUNE)
val_ds_processed = val_ds.map(preprocess_for_resnet, 
                               num_parallel_calls=tf.data.AUTOTUNE)
test_ds_processed = test_ds.map(preprocess_for_resnet, 
                                num_parallel_calls=tf.data.AUTOTUNE)
```

### 2. Chi tiết về ResNet50 Preprocessing

Hàm `preprocess_input` của ResNet50 thực hiện:
- Chuyển đổi từ RGB sang BGR (theo chuẩn Caffe)
- Trừ đi giá trị mean của ImageNet cho mỗi kênh:
  - Blue channel: mean = 103.939
  - Green channel: mean = 116.779
  - Red channel: mean = 123.68

**Công thức:**
```
image[:, :, 0] -= 103.939  # Blue
image[:, :, 1] -= 116.779  # Green
image[:, :, 2] -= 123.68   # Red
```

### 3. Tối ưu hóa pipeline

```python
# Áp dụng prefetch để tối ưu hiệu suất
train_ds_final = train_ds_processed.prefetch(tf.data.AUTOTUNE)
val_ds_final = val_ds_processed.prefetch(tf.data.AUTOTUNE)
test_ds_final = test_ds_processed.prefetch(tf.data.AUTOTUNE)
```

**Lợi ích của prefetch:**
- Tải dữ liệu song song với quá trình huấn luyện
- Giảm thời gian chờ đợi giữa các batch
- Tận dụng tối đa GPU/CPU

## VII. Tóm tắt quy trình tiền xử lý

### 1. Pipeline hoàn chỉnh

```
Dữ liệu gốc (26,000 ảnh, 26 lớp)
    ↓
Xóa lớp trùng lặp AAA (25,000 ảnh, 25 lớp)
    ↓
Tách dữ liệu (Train: 80%, Val: 10%, Test: 10%)
    ↓
Lưu dưới dạng TFRecord với nén GZIP (313 MB)
    ↓
Data Augmentation (chỉ cho Train)
    ↓
Normalization (ResNet50 preprocessing)
    ↓
Prefetch optimization
    ↓
Sẵn sàng cho huấn luyện
```

### 2. Thông số kỹ thuật cuối cùng

| Thông số | Giá trị |
|----------|---------|
| **Kích thước ảnh đầu vào** | 224 × 224 × 3 |
| **Batch size** | 32 |
| **Số lớp** | 25 |
| **Tổng số mẫu** | 25,000 |
| **Train samples** | 20,000 (80%) |
| **Validation samples** | 2,496 (~10%) |
| **Test samples** | 2,504 (~10%) |
| **Dung lượng lưu trữ** | 313 MB (TFRecord) |
| **Định dạng lưu trữ** | TFRecord + GZIP |

### 3. Ưu điểm của quy trình

✅ **Hiệu quả về dung lượng:** Giảm 45.8 lần so với lưu trữ thô

✅ **Tối ưu hiệu suất:** Sử dụng prefetch và parallel processing

✅ **Tăng tính đa dạng:** Data augmentation giúp mô hình học tốt hơn

✅ **Chuẩn hóa đúng chuẩn:** Sử dụng preprocessing của ResNet50

✅ **Dễ dàng tái sử dụng:** Dữ liệu đã được xử lý có thể dùng cho nhiều thí nghiệm

✅ **Phân bố cân bằng:** Mỗi lớp có số lượng mẫu tương đương nhau

## VIII. Kết luận

Quy trình tiền xử lý dữ liệu đã được thực hiện một cách chi tiết và khoa học, đảm bảo:

1. **Chất lượng dữ liệu:** Loại bỏ dữ liệu trùng lặp và không phù hợp
2. **Phân chia hợp lý:** Tỷ lệ 80-10-10 đảm bảo đủ dữ liệu cho huấn luyện và đánh giá
3. **Lưu trữ tối ưu:** Giảm dung lượng đáng kể mà vẫn giữ nguyên chất lượng
4. **Tăng cường dữ liệu:** Các kỹ thuật augmentation phù hợp với đặc thù của bài toán
5. **Chuẩn hóa chuẩn mực:** Sử dụng preprocessing theo chuẩn của mô hình pre-trained

Dữ liệu sau khi tiền xử lý đã sẵn sàng để huấn luyện mô hình Deep Learning với hiệu suất tối ưu.
